# Performance Summary

## Scope

- Source file: `benchmark_results.csv` generated by `benchmarks.py`.
- Visual outputs: SVG charts in `analysis_outputs/`.
- Interpretation rule: benchmark rows use different parameter tuples, so cross-scheme comparisons are directional rather than fully apples-to-apples.

## Key Findings

- The best distributed configuration is about 3.42x slower than the best minimal configuration on signing, which matches the expected overhead of two-round coordination and CRV reconstruction.
- Batched signing reduces average signing cost per message by about 70.0% relative to the best distributed single-message configuration.
- Hierarchical batching is slightly slower per message than flat batching in the best observed settings (+0.000022s), so its value is structural scalability rather than raw speed in this prototype.
- Winternitz cuts signature size to about 12.3% and communication to about 36.8% of the best distributed Lamport configuration.
- Winternitz setup is still the most expensive stage because CRV material must be prepared for every chain and every leaf.
